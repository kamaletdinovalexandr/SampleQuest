Модули:
Слой фреймворка
SCENEMANAGER - управляет сценами
UI - управляет камерой и ui сообщениями
VIEW - отображает игровые объекты, контролирует визуальное состояние
INPUT - реагирует на действия пользователя

Слой игровой логики
INTERACTION - обрабатывает игровые взаимодействия, создает события взаимодействия
INVENTORY - хранит текущие объекты
CRAFT - реагирует на взаимодействия объектов и создает текущие объекты

Проблемные места:
 - InteractionController - слишком сильно связан со многими остальными классами, уменьшить связанность
 - Item сейчас сам создает новые итемы, этим он не должен заниматься - создать отдельную сущность для крафта предметов
 - ItemView имеет те же поля что и Item, нужно заменить поля классом Item
 - Сейчас драггинг отслеживают Slot, это не его задача - нужно убрать этот функционал из него
 - Наследники ItemView имеют разные цели существования - заменить наследование базового класса на реализацию интерфейса

Возможные ошибки:
 - InteractionController: В нем сейчас взаимодействия довольно запутаны и любое изменение может приводить к ошибкам
